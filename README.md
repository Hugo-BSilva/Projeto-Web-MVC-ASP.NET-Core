## Coverage setup for SonarCloud

For SonarCloud to display coverage, you need to generate a coverage report from tests and provide that report to the scanner.

### 1) Create a test project
Run the following commands in your project folder:

```bash
dotnet new xunit -o Tests
dotnet sln add Tests/Tests.csproj
```

### 2) Add `coverlet.collector`
In the test project:

```bash
dotnet add Tests/Tests.csproj package coverlet.collector
```

### 3) Generate the coverage report in the workflow
In `.github/workflows/workflow.yaml`, add a step to run tests and generate an OpenCover file:

```yaml
- name: Run tests and generate coverage
  run: |
    dotnet restore
    dotnet test Tests/Tests.csproj --configuration Release --no-restore \
      /p:CollectCoverage=true \
      /p:CoverletOutputFormat=opencover \
      /p:CoverletOutput=Tests/TestResults/coverage/
```

### 4) Configure SonarCloud to read the report
Add or update the SonarCloud step in the workflow with the report path:

```yaml
- name: SonarCloud Scan
  uses: SonarSource/sonarqube-scan-action@v5.0.0
  env:
    SONAR_TOKEN: ${{ secrets.SONAR_TOKEN }}
  with:
    args: >
      -Dsonar.cs.opencover.reportsPaths=Tests/TestResults/coverage/coverage.opencover.xml
```

### 5) Optional: add it to `sonar-project.properties`
If you are already using `sonar-project.properties`, also include this line:

```properties
sonar.cs.opencover.reportsPaths=Tests/TestResults/coverage/coverage.opencover.xml
```

### 6) Expected result
After this, SonarCloud should show coverage in the project `Coverage` tab using the `coverage.opencover.xml` file generated by `dotnet test`.

> Note: without tests and a coverage report, SonarCloud cannot calculate coverage automatically.
